# 封装全局接口请求

帮我完成项目的全局接口请求封装，只实现请求层，不需要业务页面。如果已有全局接口封装，请按照以下的文件，检查优化

---

# 接口返回结构

项目所有接口返回结构固定如下：

```json
{
  "ResultData": {},
  "IsSuccess": true,
  "ErrorCode": 0,
  "ErrMsg": null,
  "RedirectUrl": null,
  "Version": null
}
```

---

# 接口返回处理规则

## 1. 成功处理

当：

```js
IsSuccess === true;
```

表示接口请求成功。

真正的业务数据在：

```js
ResultData;
```

request 封装层需要统一处理：

```js
return ResultData;
```

调用层拿到的直接就是业务数据。

例如：

```js
const data = await loginApi(params);

console.log(data);
```

而不是：

```js
res.ResultData;
```

---

## 2. 失败处理

当：

```js
IsSuccess === false;
```

表示接口请求失败。

request 层统一处理：

- Toast 提示：

  ```js
  ErrMsg;
  ```

- Promise reject

例如：

```js
toast(ErrMsg);

return Promise.reject(response);
```

调用层不需要重复判断：

```js
IsSuccess;
```

# request 全局封装要求

## 功能要求

### 1. 支持请求方式

需要支持：

- GET
- POST
- PUT
- DELETE

---

### 2. 自动处理

统一自动处理：

- loading 状态
- 接口错误提示
- 网络异常提示
- 超时提示
- HTTP 状态码异常

---

### 3. 统一配置

支持统一配置：

```js
baseURL;
timeout;
headers;
```

---

### 4. token 自动注入

请求发送时自动携带 token。

例如：

```js
Authorization: Bearer token
```

token 从：

```js
localStorage;
```

或项目统一缓存中获取。

---

### 5. 支持拦截器

需要实现：

- 请求拦截器
- 响应拦截器

---

### 6. 简洁调用方式

调用方式尽量简单：

```js
request.get("/user/info");

request.post("/login", data);
```

---

### 7. request 层统一处理返回结构

统一处理：

```js
IsSuccess;
ErrMsg;
ResultData;
```

---

### 8. response 拦截器处理规则

成功：

```js
return ResultData;
```

失败：

```js
toast(ErrMsg);

return Promise.reject(response);
```

---

### 9. 工程化要求

代码要求：

- 不要伪代码
- 要完整可运行
- 注释简洁清晰
- 结构清晰
- 方便后续维护
- 尽量工程化
- 不要写页面
