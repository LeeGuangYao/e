# 0001 - 自研 i18n 方案（不使用 vue-i18n）

- 状态：已采纳
- 日期：2026-05-08

## 上下文

项目需要支持多语言（中文、英文、繁体、日文），需要选择 i18n 方案。

## 决策

自研轻量级 i18n 方案，不使用 vue-i18n。

## 理由

1. **体积**：vue-i18n 较重，商旅系统只需要基本的 key-value 翻译 + 参数插值，不需要其消息格式化（ICU MessageFormat）、复数规则、日期/数字本地化等高级功能
2. **灵活性**：自研方案可直接集成 `localStorage.CurrentLanguage` 和 URL query `LanguageType` 的读取逻辑，与 useAppStore/useUserStore 无缝对接
3. **TypeScript**：自研方案通过 `typeof zh` 导出 `LocaleMessages` 类型，编译期即可检测所有语言文件的 key 是否完整一致
4. **简单性**：`$t(key)` + 参数插值 `{name}` 即可满足 B 端系统需求，无需引入额外依赖

## 后果

- 优点：零额外依赖、编译期类型安全、完全可控
- 缺点：需要自行维护 i18n 工具类、不兼容 vue-i18n 生态插件
